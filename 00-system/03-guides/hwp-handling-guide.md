# 한글(HWP/HWPX) 문서 다루기 가이드

> **준비물**: Claude Code 설치 완료, do-better-workspace 클론 완료

---

## 왜 이게 필요한가요?

공공기관·학교·공공조직과 일하면 한글 파일이 끝없이 나옵니다. 공문, 보고서, 견적서, 신청서, 회의록. 여기에 두 가지 답답함이 있습니다.

| 상황 | 기존 방식 | 이 가이드의 방식 |
|------|----------|----------------|
| 한글 공문 10개 받아서 핵심만 뽑아내기 | 일일이 열어서 읽고 복사 | AI에게 "이 폴더 한글 파일들 정리해줘" |
| 마크다운으로 작성한 보고서를 한글 양식으로 | 복사 → 한글 붙여넣기 → 서식 재조정 | "이 한글 양식에 이 내용 넣어줘" |
| 학교마다 다른 한글 양식 53개 채우기 | 사람이 하루 종일 | 양식 1개 분석 → 53개 자동 생성 |

이 가이드는 한글 파일을 **읽는 도구(kordoc)** + **쓰는 도구(hwpxskill)** 두 개를 어떻게 같이 쓰는지 설명합니다.

---

## 두 도구의 역할 차이

### kordoc — 한국어 문서를 마크다운으로 읽어들이는 도구

- **읽기 전문**: HWP, HWPX, PDF, XLSX, DOCX 등 한국 사무실에서 굴러다니는 거의 모든 문서 포맷을 마크다운으로 변환
- **AI가 이해하기 쉬운 상태**로 만들어줌 — 표 구조, 셀 병합까지 살려서
- **MCP 서버**로 자동 등장 (Claude Code 시작하면 자동 연결)
- 한컴오피스 설치 불필요

### hwpxskill — 한글(HWPX) 문서를 만들거나 편집하는 도구

- **쓰기/편집 전문**: 기존 HWPX 양식을 분석해서 구조를 보존한 채 내용만 갈아끼우기
- **양식 한 개를 레퍼런스로 두고 여러 개 자동 생성** (53개 학교 양식 같은 케이스)
- Claude Code **스킬**로 자동 등장 (HWPX 파일 작업 시 알아서 호출됨)
- HWPX만 지원 (구버전 .hwp는 안 됨 — 한글 2010 이후는 모두 HWPX로 저장 가능)

### 함께 쓰는 흐름

```
한글 공문/양식 받음 (HWP/HWPX)
       ↓ kordoc
   마크다운 (AI가 분석)
       ↓ Claude가 내용 작성/수정
   새 마크다운
       ↓ hwpxskill
   원본 양식 그대로 + 내용만 바뀐 HWPX
```

## 시나리오 1: 한글 공문을 마크다운으로 읽어들이기

받은 HWP/HWPX 파일을 AI가 분석할 수 있는 상태로 만듭니다.

### 대화 예시

> 이 한글 파일 읽어줘: `~/Downloads/회의록.hwp`

Claude가 자동으로 kordoc을 호출해서 텍스트·표·헤딩을 그대로 살려 마크다운으로 변환합니다. 결과를 그 자리에서 분석·요약·검색에 바로 사용 가능.

### 직접 명령으로도 가능

터미널에서:
```bash
npx -y kordoc "파일경로.hwp" > result.md
```

### 처리 가능한 형식

- HWP 3.x (1996~2002년 구버전)
- HWP 5.x (오늘날 대부분)
- HWPX (한글 2010 이후 기본 포맷)
- PDF, XLS, XLSX, DOCX

복잡한 표(셀 병합, rowspan/colspan)도 마크다운 표로 정확히 재현.

## 시나리오 2: 마크다운 보고서를 한글 양식으로 출력

AI와 작성한 마크다운을 한글 양식에 그대로 채워 넣습니다.

### 준비

기관 표준 양식 HWPX 파일 1개를 워크스페이스 어딘가에 두세요. 예: `~/Downloads/보고서_양식.hwpx`

### 대화 예시

> `~/Downloads/보고서_양식.hwpx` 양식을 그대로 유지하면서 아래 내용으로 채워줘.
>
> [본문 내용 붙여넣기]

Claude가 hwpxskill을 호출해서:
1. 원본 양식의 글자 모양, 표 구조, 페이지 여백, 머리말·꼬리말 분석
2. 본문만 새 내용으로 교체
3. 페이지 수 변화가 있는지 자동 검사

결과물은 새 HWPX 파일로 저장. 한글에서 열면 원본 양식 그대로 + 내용만 바뀐 상태.

## 시나리오 3: 양식 1개로 N개 문서 자동 생성

학교 53개 양식 같은 케이스. 양식 1개 + 데이터 시트 1개 → 자동으로 N개 HWPX 생성.

### 준비

- 양식 파일 1개 (예: `참여확인서_양식.hwpx`)
- 데이터 시트 (Google Sheets, CSV 등 — 이름·날짜·기관명 등)

### 대화 예시

> `participants.csv`에 있는 53명에 대해서 `참여확인서_양식.hwpx`를 채워서 각각 별도 파일로 저장해줘. 이름 형식은 `2026-05-14_참여확인서_{이름}.hwpx`로.

Claude가:
1. CSV에서 53행 읽기
2. 양식 HWPX 1번 분석
3. 53번 반복하면서 이름·날짜만 바꿔 새 파일 생성

사람 손이 가는 부분: 시트 1번, 양식 1번. 끝.

## 시나리오 4: 한글 ↔ 한글 비교 (신구대조표)

A안과 B안 비교, 회의록 수정 전후 비교에 씁니다.

### 대화 예시

> 이전 회의록과 새 회의록의 차이를 신구대조표로 만들어줘:
> - 이전: `회의록_v1.hwp`
> - 새 버전: `회의록_v2.hwp`

kordoc이 양쪽을 마크다운으로 읽어들이고, Claude가 줄 단위로 무엇이 추가·삭제·수정됐는지 정리.

## 자주 만나는 문제

### Q1. "이 파일을 읽을 수 없습니다"

- **DRM 걸린 한글 문서일 가능성**: 일부 공공기관 배포 한글 파일은 DRM 보호. 한컴오피스에서 한 번 열어 저장하면 풀림
- **암호 걸린 파일**: 한글에서 암호 해제 후 다시 시도

### Q2. 변환 결과에 깨진 글자가 보임

- 거의 대부분 HWP 3.0 구버전 한자/특수기호 케이스. 본문 텍스트는 정상이고 일부 특수기호만 영향
- 깨진 부분이 핵심이면 한글에서 직접 확인

### Q3. hwpxskill로 만든 파일이 한컴오피스에서 안 열림

- 거의 없지만 발생하면 page_guard 결과를 보고 원본 양식과 어떤 차이가 나는지 확인
- 임시 해결: 한글에서 "새로 만들기" → 본문만 복사

### Q4. HWP는 되는데 HWPX는 어떻게 만드나

- 한글에서 파일 → 다른 이름으로 저장 → 형식 HWPX 선택
- 한글 2010 이후라면 모두 가능

## 한 줄 요약

받은 한글 파일을 AI가 읽으려면 **kordoc**, AI가 한글 파일을 만들려면 **hwpxskill**. 두 개를 같이 두면 한글 문서 작업 흐름 전체가 자동화됩니다.
